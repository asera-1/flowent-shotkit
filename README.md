# flowent-shotkit

Turn raw app screenshots into store-ready **App Store / Google Play** screenshot kits —
a **deterministic renderer** plus an optional **AI director** for the copy.

One TypeScript engine, runs in Node (CLI/CI) and in the browser. No backend.

## ▶ Run the playground (web studio)

The fastest way to use this is the **studio** — a local web app where you upload
screenshots, edit headlines (or auto-write them), pick a theme + target stores, see a
**live preview**, and export the whole kit as a ZIP. Everything runs in your browser; no
backend and no API key required.

**Windows — double-click [`play.cmd`](./play.cmd).** It installs everything and opens the
studio for you.

**Any OS — terminal:**

```bash
npm install     # installs the engine AND the studio in one step (npm workspaces)
npm run dev      # opens http://localhost:5173
```

> Tip: if `npm` "isn't recognized" on Windows, Node isn't on your PATH. Use `play.cmd`
> (it adds Node automatically), or install Node LTS from https://nodejs.org.

What you can do in the studio: upload one or more screenshots · edit the headline per
slide · **Auto-headlines** (offline) or bring your own AI key for full vision writing ·
switch theme presets (Brand Blue, Iridescent, Teal, Clean Light, Mesh) · toggle the
device-frame template · A/B copy variants · localize copy · preview every store size ·
**Export ZIP** (organized by store, with a contact sheet).

## Headless / CI rendering

Prefer config-driven batch rendering (no UI)? The same engine runs from the CLI:

```bash
npm install
npm run render -- --config shotkit.config.json --out out
```

Outputs, organized by store:

```
out/manifest.json
out/app-store/apple-iphone-69/01-home.png   (1290×2796)
out/app-store/apple-ipad-13/01-home.png     (2064×2752)
out/google-play/play-phone/01-home.png      (1080×1920)
```

## Why

A store screenshot is a composition of known layers (background → device → screen →
status bar → headline). Once each layer is generated programmatically, producing dozens
of polished, on-brand slides is a pipeline, not a design sprint. See
[`docs/ENGINE.md`](./docs/ENGINE.md) for the full method and
[`docs/REUSABILITY.md`](./docs/REUSABILITY.md) for the architecture.

## Config

```jsonc
{
  "storeIds": ["appstore-iphone-69", "appstore-ipad-13", "play-phone"],
  "theme": {
    "fontFamily": "Montserrat", "weightLine1": 600, "weightLine2": 700,
    "headlineColor": "#F8FAFF", "gradient": ["#AEE4FF", "#F4FBFF"],
    "background": { "kind": "synthetic", "from": "#5CA8FF", "to": "#1C46DE", "glow": "rgba(150,200,255,0.30)", "grain": 7 }
  },
  "slides": [
    { "id": "01-home", "screenshot": "assets/examples/home.png", "headline": { "line1": "YOUR NEXT LESSON", "line2": "READY EVERY DAY" } }
  ]
}
```

## Programmatic

```ts
import { renderProject, allTargets, DeterministicDirector } from 'flowent-shotkit'
```

## The AI Director (optional)

The renderer is 100% local. The **Director** is the only AI piece: it reads screenshots
and writes the headline set + order + theme into the config. It runs once per project,
never per render. Ships with a no-API `DeterministicDirector` and a BYOK `LlmDirector`
interface. See `docs/REUSABILITY.md` §12.

## Project layout

```
src/                 engine: detect, targets, background, device, statusbar,
                     headline, compose, template, contactsheet, director/*
  node/canvas-target.ts   Node canvas adapter (@napi-rs/canvas)
studio/              the web playground (Vite + React) — imports the engine via @engine
assets/fonts         assets/examples
```

## Scripts (run from the repo root)

- `npm run dev` — start the studio playground (http://localhost:5173)
- `npm run build:studio` — production build of the studio
- `npm run render` — render a config to PNGs + manifest (headless)
- `npm run typecheck` — `tsc --noEmit` (engine)
- `npm test` — unit tests (Vitest)
- `npm run build` — bundle the engine to `dist/` (tsup)
